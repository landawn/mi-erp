
**问题来了：Anthropic 是如何“直接”向 Broadcom 下单购买 TPU 的？**

一种合理的推测是：**Anthropic可能是在Google的授权或协调下，确实直接向Broadcom下单采购TPU相关系统，并最终在一定意义上“拥有”这些 TPU**。尽管各方并未披露合同细节，但这并不妨碍我们基于产业逻辑进行推断。

### 一、Anthropic 向 Broadcom 采购的，很可能不是“裸TPU”，而是整机系统

Anthropic 与 Broadcom 的 **210 亿美元合同**，大概率并非采购零散的 TPU 芯片，而是**包含 TPU 在内的完整系统**（例如以机柜为单位的整机交付）。

如果只是购买“散装TPU”，再Google 的工程团队将这些TPU插到主板上，这在工程和责任划分上都非常不合理，也没有必要。因此，更合理的形式是：

* Broadcom 交付 **按机柜封装好的完整 TPU 系统**
* Anthropic 将这些系统部署在 **Google 的数据中心**
* 接通电源和网络即可使用

### 二、为什么 Anthropic 不直接以“云服务”的方式向 Google 购买 TPU？

可能有几个现实原因：

1. **财务 / 会计 / 税务结构不同**
   直接采购并资本化硬件，与按云服务付费，在财务报表、税务处理和长期成本结构上可能存在显著差异。

2. **长期成本更低**
   如果 Anthropic 自己出资购买并拥有整套系统，它只需向 Google 支付：

   * 数据中心机房费用
   * 电力费用
   * 可能还有一定的 TPU 使用 / 管理费用

   这种模式在长期看，**可能显著低于按需从 Google Cloud 购买 TPU 服务的成本**。

3. **Google 的战略考量**
   Google 之所以同意这种模式，很可能出于战略原因：

   * Anthropic / Claude 的长期潜力巨大，但当前收入规模尚不足以支撑极高的云账单
   * 允许其自购硬件，相当于 **变相“补贴”或扶持 Anthropic**
   * 同时又能 **绑定长期合作关系**

   这与 **NVIDIA 对 OpenAI 及其他核心合作伙伴的策略非常相似**：
   从自身高利润中拿出一部分（比如 10%–20%），扶持生态伙伴，共同推动整个 AI 产业的发展。

### 三、从“百万 TPU”目标反推成本结构

根据公开信息，Google 与 Anthropic 的合作中提到：

> *“Anthropic will train and serve Claude models with plans to access up to 1 million TPUs, worth tens of billions of dollars.”*

同时，行业估算显示：

> *一个 1GW 数据中心的成本约为 500 亿美元，其中约 350 亿美元用于芯片。*

据此推算：

* 如果部署 **100 万颗等效算力的芯片**
* 使用 NVIDIA GPU，芯片成本可能在 **300 亿美元左右**
* Google 自研 TPU 在成本上应有明显优势，**假设打折后约 200 亿美元**

那么问题来了：
**Anthropic 是否用 210 亿美元就从 Broadcom 买下了 100 万颗 TPU？**

大概率不是。

更合理的解释是：

* 210 亿美元采购的是 **按机柜交付的一整套 TPU 系统**
* 规模可能在 **几十（比如50）万 TPU 左右**
* 剩余算力需求，则通过 **Google Cloud 的形式** 从 Google 获得（且很可能享有大幅折扣）

### 四、一种可能的合作结构（推测）

综合以上因素，一个合理的协议结构可能是：

1. **硬件采购与部署**

   * Google 允许 Anthropic 直接从 Broadcom 采购一定数量（例如 50 万 TPU 规模）的整机系统
   * 这些系统部署在 Google 的数据中心
   * Anthropic 向 Google 支付机房、电力及相关管理费用，并可能支付一定的 TPU 使用费

2. **算力优先权与排他性**

   * 对于剩余的算力需求，Google 拥有 **优先供给权**
   * 在双方约定的算力范围内（最高至 100 万 TPU），Anthropic 的新增需求应优先从 Google 获取
   * 只有在超过约定上限，或 Google 无法满足需求时，Anthropic 才能考虑从其他云厂商（如 AWS、Microsoft）采购

3. **非强制使用条款**

   * 如果 Anthropic 的算力需求低于协议上限，它**并不需要强制从 Google Cloud 购买 TPU 服务**
   * “100 万 TPU”更多是一个 **优先权与上限约定**，而非强制消费承诺

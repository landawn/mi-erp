# **结论：**

1）**几乎铁定：** Oracle 下半年会部署 AMD 最新的 AI 芯片，这些芯片由 OpenAI 从 AMD 购买；Oracle 不需要出钱。AMD 很可能给 OpenAI 一部分投资，用于购买芯片。这批芯片可能价值四五百亿美元，但预计 OpenAI 最终只需支付两百亿左右给 AMD。

2）OpenAI 与英伟达的“千亿合作”在 26 年很可能不会有进一步消息。

3）现有的算力合作足以支撑 OpenAI 到下半年 AMD 新芯片上线。

4）大概率 OpenAI 下半年某个时候会和博通说再见，它的自研芯片计划大概率取消或延迟。

# **ChatGPT、Gemini、Claude 的用户规模：**

• ChatGPT 接近 10 亿月活
• Gemini 大概五六亿
• Anthropic 只有两三千万

**Anthropic 的算力来源：**
* 除了现有算力外，Amazon 还有一个新的数据中心专门给它（Project Rainier：印第安纳州价值 110 亿美元的大型数据中心）
* 与谷歌与高通的合作：下半年 200 亿规模的 TPU
* 与英伟达和微软 300 亿的合作
* 单从模型本身看，Claude 的 LLM 相较 ChatGPT 和 Gemini 还是有一定差距，只是它的 Claude Code 客户端做得非常好。
* OpenAI 和 Gemini 在编程方面越做越好，会抢走或限制 Claude Code 的发展速度。

基于 Anthropic 的用户规模，上述算力足以支撑它 2026/27 的发展，所以 2026 年它不太可能有新的大合作协议，最多是现有协议的扩容。

# **博通（Broadcom）：**

博通的前三大客户：谷歌、Meta、字节跳动。
* 谷歌订单属于常规
* Meta 订单仍是常规
* 字节跳动（传闻会采购大量英伟达 H200，如果成真，势必会减少自研芯片的采购）
* Anthropic 的订单是一次性的
* 如果 Meta 采购谷歌的 TPU，它势必会减少自己的订单，一增一减，对博通不是好消息。（如Meta把TPU部署在自己的数据中心，它为每个TPU支付的价格将显著高于Anthropic的购买的价格）

**一句话：博通大概率到顶了。**

# **Oracle：**

如果三方（OpenAI、AMD、Oracle）能够实现“星际之门”到 2029 年底 5000 亿投资的计划，按照目前的数据中心成本结构：
这 5000 亿中至少有 2000 亿与 GPU 直接相关。

如果 GPU 供应由 OpenAI 和 AMD 主导：
* Oracle 仍需自掏腰包投入 2000 亿,贷款光利息估计在 100 亿以上
* 还需要至少融资 1000 亿
* 2000 亿美元的 GPU，如果按五年的摊销成本，每年就是 400 亿
* 如果到 2030 年这 5000 亿规模的数据中心全部上线，Oracle / OpenAI 每年为此需要支付的成本估计将达到 1000 亿美元。OpenAI 至少需要拿出 1500 亿美元的营收来维持其数据中心的支出。

以它当前的营收 / 利润 / 现金流规模，Oracle再融资 1000 亿将非常困难。

**它的命运完全系在 OpenAI 的裤腰带上。**
